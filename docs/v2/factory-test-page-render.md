# FactoryTest 页面渲染说明

本文根据 `main/display/pages/factory_test_page_adapter.cc` 的布局代码整理，描述工厂测试页（FT 页面）在 400x300 墨水屏上的实际视觉结构与动态变化。

## 1. 页面尺寸与分区

- 画布尺寸：`400 x 300`
- 顶部头部（Header）：高度 `48`
- 底部说明（Footer）：高度 `24`
- 左侧步骤列（Step Panel）：宽度 `104`
- 右侧主内容区（Main Panel）：宽度 `296`（`400 - 104`）
- 步骤行数：`7` 行
- 每行高度：`24`

## 2. 视觉结构（示意图）

```text
+------------------------------------------------------------------------------+
| [FT测试]                                                        [1/7] [WAIT] |
|------------------------------------------------------------------------------|
| +--------------------+ +---------------------------------------------------+ |
| | RF             >   | | RF 测试                                           | |
| | 音频                | | 正在扫描目标 Wi-Fi                                | |
| | RTC                 | | SSID=Yoxintech                                    | |
| | 充电                | | RSSI=-63 dBm                                      | |
| | LED                 | | 连续命中=2/3                                       | |
| | 按键                | | 阈值=-70 dBm                                      | |
| | NFC                 | |                                                   | |
| +--------------------+ +---------------------------------------------------+ |
| 全部通过，请拔掉 USB 后按确认                                                   |
+------------------------------------------------------------------------------+
```

说明：

- 左列每一行是独立容器（非 `lv_table`），左侧为步骤名，右侧为状态符号。
- 当前步骤会高亮反色（黑底白字）。
- 步骤状态图标规则：`>` 运行中、`X` 失败、空白=等待或通过（通过主要靠步骤名删除线体现）。

## 3. 各区域内容来源

### Header

- 左侧标题固定：`FT测试`
- 右侧步进：`1/7 ... 7/7`
- 最右状态：`WAIT / RUN / PASS / FAIL`

### 左侧步骤列表

固定 7 项（按顺序）：

1. RF
2. 音频
3. RTC
4. 充电
5. LED
6. 按键
7. NFC

### 右侧主内容区

包含 6 行文本位：

- `title`：当前测试标题（如“RF 测试”）
- `hint`：提示语（如“正在扫描目标 Wi-Fi”）
- `detail1~detail4`：每个测试步骤的实时细节

### Footer

- 底部全宽提示，常用于“关机动作提醒”或“通过后操作引导”。

## 4. 动态刷新机制（你看到“画面变化”的来源）

- 业务侧 `FactoryTestService` 持续更新 `FactoryTestSnapshot`。
- UI 侧 `FactoryTestPageAdapter::UpdateSnapshot()` 把 snapshot 写入各个 label。
- 调用 `display_->RequestUrgentRefresh()` 触发墨水屏紧急刷新。
- 墨水屏驱动内部根据差异策略选择全刷或局刷。

## 5. 按键测试步骤的特殊视觉

当步骤进入“按键测试”时，`detail1~detail3` 支持前缀语法：

- `[>] 文本`：当前执行项（黑底白字）
- `[x] 文本`：已完成项（删除线）
- `[X] 文本`：失败项（前面显示 `X`）

典型显示效果如下：

```text
确认键：已通过（删除线）
上键：当前项（黑底白字）
下键：等待
```

## 6. 结论

该 FT 页本质是“**手工布局的卡片+列表页面**”，不是表格控件。  
你看到的“表格感”主要来自左侧 7 行步骤框和统一边框样式。
